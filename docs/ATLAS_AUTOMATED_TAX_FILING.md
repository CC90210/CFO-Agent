## 1. CRA NETFILE / EFILE — How It Actually Works

### What Is NETFILE?

NETFILE is the CRA's system for **individuals** to file their own T1 personal income tax returns electronically. You use CRA-certified tax software, which generates a `.tax` file, and that file is transmitted to the CRA over the internet.

- **URL:** https://www.canada.ca/en/revenue-agency/services/e-services/digital-services-individuals/netfile-overview.html
- **Who uses it:** Individual taxpayers filing their own returns
- **File format:** `.tax` files generated by certified software (proprietary binary/XML format)
- **Transmission:** Software connects to CRA servers via an API and transmits the `.tax` file
- **Authentication:** SIN + date of birth + prior-year data (varies by software)

### What Is EFILE?

EFILE is the CRA's system for **tax professionals** (accountants, tax preparers) to file returns on behalf of clients.

- **URL:** https://www.canada.ca/en/revenue-agency/services/e-services/e-services-businesses/efile-electronic-filers.html
- **Who uses it:** Registered tax preparers with an EFILE number
- **Requirements:** Must register with CRA, pass suitability screening, renew annually
- **Client authorization:** Client must sign form T183 before preparer can EFILE
- **Mandatory:** Since January 1, 2024, preparers filing >5 returns/year MUST e-file. Penalty: $25/T1 or T3 paper-filed, $100/T2 paper-filed.

### Key Difference: NETFILE vs EFILE

| Feature | NETFILE | EFILE |
|---------|---------|-------|
| **User** | Individual taxpayer | Registered tax professional |
| **Registration** | None needed | Must apply for EFILE number |
| **Authorization** | Self-filing | Client signs T183 |
| **Software** | Consumer tax software | Professional tax software |
| **Return types** | T1 only | T1, T2, T3 |
| **Batch filing** | No | Yes (via professional software) |

### Technical Protocol

**The CRA does NOT have a public REST API.** The filing mechanism works as follows:

1. Certified tax software calculates the return and generates a `.tax` file
2. The software connects to CRA's NETFILE/EFILE servers (proprietary protocol)
3. The `.tax` file is transmitted (this is essentially XML wrapped in a proprietary format)
4. CRA validates the submission and returns a confirmation number
5. For information returns (T4, T5, etc.), CRA accepts raw XML via Internet File Transfer (IFT)

**Key technical details:**
- The CRA's NETFILE system uses an **API** that certified vendor software connects to — this was confirmed by the Canadian Digital Service (CDS) documentation
- The `.tax` file format is **not publicly documented** — it is only available to certified software developers under NDA/confidentiality agreement
- XML schemas for **information returns** (T4, T5, T4A, etc.) are publicly available at: https://www.canada.ca/en/revenue-agency/services/e-services/filing-information-returns-electronically-t4-t5-other-types-returns-overview/xml-specs.html
- Schema download: `xmlschm1-26-3.zip` (180 KB, version 1-26-3, updated December 2025)
- Maximum XML file size: 150 MB (1 GB uncompressed)
- Starting 2025: All returns in a submission must be same type (no mixed returns)
- Starting January 2025: Most submissions validated in real time at return level

### The CDS Open-Source Proof of Concept

**Critical finding:** The Canadian Digital Service (CDS) built an open-source prototype called **"Claim Tax Benefits"** that files returns through the NETFILE API.

- **GitHub:** https://github.com/cds-snc/cra-claim-tax-benefits
- **Documentation:** https://cds-snc.github.io/claim-tax-benefits-documentation/technical-overview/
- **Architecture:** Node.js/Express app with Pug templates
- **How it works:** The product communicates with the CRA database via an API, retrieves individual data, confirms with user, and files through the NETFILE API — the same one used by vendor software
- **Status:** Archived (prototype only), but proves the NETFILE API can be accessed programmatically

> **ATLAS NOTE:** This is the single most important finding. The CDS prototype proves that the NETFILE API exists, is accessible, and can be called from custom software. The barrier is CRA certification, not technical impossibility.

## 2. QuickBooks Online Accountant + ProFile

### ProFile (Intuit Canada — Desktop Software)

- **URL:** https://profile.intuit.ca/
- **What it does:** Professional tax preparation for T1, T2, T3 returns with EFILE
- **EFILE support:** Yes — T1, T2, T3 (requires CRA EFILE number)
- **HST/GST:** Not directly filed through ProFile (use CRA My Business Account or separate GST software)
- **NETFILE:** Cannot NETFILE from Pro Tax (use TurboTax for that)

**Pricing (approximate — visit https://profile.intuit.ca/pricing/ for current):**
- License entitles **unlimited returns** per licensed location
- Free trial: Up to 7 T1, T2, and T3 returns per firm
- Requires active QuickBooks Online Accountant (QBOA) account for Pro Tax
- Subscription only needed when you EFILE, print, or export

**Automation/API:**
- **No public API** — ProFile is GUI-only desktop software
- Cannot be scripted or automated programmatically
- Can import from prior-year ProFile files and some competitor formats
- Pro Tax (cloud version) requires QBOA account

### QuickBooks Online Accountant (QBOA)

- **URL:** https://quickbooks.intuit.com/ca/accountants/
- **Pricing:** Free for accountants (clients pay for QBO subscriptions)
- **Pro Tax module:** Built into QBOA, supports T1, T2 filing
- **API:** QuickBooks Online has a REST API for accounting data, but **not for tax filing**
- The QBO API handles invoices, expenses, bank feeds — not CRA submissions

**Bottom line:** ProFile/Pro Tax is GUI-only. No API. Cannot automate tax filing through it.

## 4. Programmatic Tax Filing Options

### Option A: Become a Certified Software Developer (The Real Path)

The CRA has a formal process for certifying tax preparation software:

1. **Express interest** to CRA's software developer program
2. **Sign confidentiality agreement** to access draft forms and specifications
3. **Receive test returns** (mid-November each year) from CRA
4. **Build software** that generates compliant `.tax` files / XML
5. **Transmit test returns** to CRA for evaluation
6. **Receive certification letter** if software passes — can then market as NETFILE/EFILE certified
7. **Renew annually** — specifications change each tax year

**What CRA tests for:**
- Correct tax calculations
- Proper XML/file format generation
- Successful transmission to CRA systems
- CRA does NOT test for UX, spelling, or grammar

**Key URLs:**
- EFILE certified software list: https://www.canada.ca/en/revenue-agency/services/e-services/digital-services-individuals/efile-electronic-filers/efile-certified-software-efile-program.html
- Apply for EFILE: https://www.canada.ca/en/revenue-agency/services/e-services/digital-services-individuals/efile-electronic-filers/apply-efile.html

### Option B: Use Professional Software with Batch Operations

**TaxCycle (by Xero):**
- **URL:** https://www.taxcycle.com/
- **Cost:** $2,440/user/year for complete suite (T1, T2, T3, T4, T5, T5013, etc.)
- **Batch filing:** Yes — Client Manager supports batch T1 EFILE, T1013, T1135, T2 Corporation Internet Filing, and more
- **Carryforward:** Batch carryforward from prior year
- **Import:** Can carry forward from DT Max, Cantax, Taxprep, ProFile
- **Integrations:** Xero, QuickBooks, DoxCycle (document management)
- **API:** No public REST API — but batch operations through desktop Client Manager
- **Best for:** Medium-to-large practices doing bulk filing

**Taxprep (by Wolters Kluwer):**
- **URL:** https://www.wolterskluwer.com/en-ca/solutions/taxprep
- **Cost:** CCH iFirm Taxprep T1 from $725, T2 from $1,293, T1+T2 bundle from $1,902
- **Used by:** Every single Big 4 firm in Canada + thousands of local firms
- **Cloud version:** CCH iFirm Taxprep (Canada's first cloud-based tax prep)
- **Batch filing:** Yes (enterprise-level)
- **API:** Not publicly documented — likely has internal APIs for enterprise clients
- **Best for:** Large firms, Big 4, enterprise batch filing

### Option C: The CDS Open-Source Approach

As documented in Section 1, the Canadian Digital Service proved that:
- The NETFILE API is real and accessible
- A Node.js application can communicate with it
- The same API used by vendor software can be called from custom code
- **BUT:** You need CRA certification to use it in production

**GitHub:** https://github.com/cds-snc/cra-claim-tax-benefits

### Option D: AI-Powered Tax Tools (Emerging)

**TaxGPT (https://taxgpt.ca/):**
- AI agent that logs into your existing tax software (Axcess, UltraTax, Drake, ProConnect)
- Does NOT file directly — automates the GUI of existing software
- Currently US-focused with Canadian capabilities

**CloudTax (https://www.cloudtax.ca/):**
- Free Canadian tax filing
- AI-powered OCR for slip extraction
- NETFILE-certified
- No API — consumer product

**CanTax.ai (https://cantax.ai/):**
- AI-driven Canadian tax advice
- Not a filing tool — advisory only

### How Do Large Firms Batch-File Thousands of Returns?

**The Big 4 (KPMG, Deloitte, PwC, EY) use:**
1. **Taxprep** (Wolters Kluwer) — the dominant professional platform in Canada
2. **Proprietary internal tools** — each firm has custom platforms (Deloitte: Omnia, KPMG: Clara + tax toolsets)
3. **Batch EFILE through Client Manager** — TaxCycle and Taxprep both support batch transmission
4. **The workflow:** Data entry/import into professional software -> review -> batch EFILE to CRA
5. **They do NOT use a public API** — they use EFILE-certified professional software with batch capabilities

## 5. The EFILE Program — Registration & Certification

### Can OASIS AI Solutions Register as an EFILE Service Provider?

**Yes, but with conditions:**

**Eligibility (from CRA):**
- Any firm, organization, or individual providing tax preparation services can apply
- Must be a "tax preparation service" — legal form doesn't matter (sole prop, corporation, etc.)
- Employees who prepare taxes as part of their job duties are NOT eligible (must be a service provider)
- Must pass CRA suitability screening (background check, ~30 business days)
- No fee for registration

**URL:** https://www.canada.ca/en/revenue-agency/services/e-services/digital-services-individuals/efile-electronic-filers/apply-efile.html

**Requirements:**
- Apply online (no paper applications)
- Receive EFILE number and password
- Renew annually
- Use EFILE-certified software
- Have clients sign T183 form
- Maintain records per CRA requirements

### Can ATLAS Register as Certified NETFILE/EFILE Software?

**Yes, in theory. Here's the process:**

1. **Contact CRA** to express interest in software certification
2. **Sign confidentiality agreement** (NDA to access draft forms and specs)
3. **Receive T1 EFILE specification** (the `.tax` file format, XML schema, validation rules)
4. **Build software** that generates compliant returns
5. **Mid-November:** CRA sends test returns for the upcoming tax year
6. **Transmit test returns** to CRA's testing environment
7. **CRA evaluates** and issues confirmation letter if passes
8. **Market as certified** NETFILE/EFILE software

**Gotchas:**
- Must recertify every single year (tax law changes annually)
- Must sign confidentiality agreement — T1 spec may not be fully open-sourceable
- CRA testing infrastructure has limited windows
- Need to handle ALL tax scenarios for the forms you support
- Ongoing maintenance burden is significant

### Is This Realistic for ATLAS?

**Assessment:** The certification path is real but resource-intensive. A more practical approach:

1. **Short term:** Use TaxCycle ($2,440/yr) with batch operations for CC's returns
2. **Medium term:** Register OASIS as EFILE service provider, use certified software
3. **Long term:** If ATLAS becomes a product, pursue CRA software certification
4. **Sellable angle:** ATLAS doesn't need to BE certified software — it can be an AI layer that prepares data and feeds it into certified software (like TaxGPT does)

## 6. US Comparison — IRS MeF, Direct File, IRIS

### IRS Modernized e-File (MeF)

The IRS is significantly more open than CRA for programmatic filing.

**Overview:**
- **URL:** https://www.irs.gov/e-file-providers/modernized-e-file-overview
- **Format:** XML (publicly documented schemas)
- **Schemas:** https://www.irs.gov/e-file-providers/modernized-e-file-mef-schemas-and-business-rules
- **Two channels:**
  - **IFA (Internet Filing Application):** Browser-based
  - **A2A (Application to Application):** Programmatic SOAP web service
- **Protocol:** SOAP 1.1 with attachments over HTTPS
- **Authentication:** Username + X.509 digital security certificate from IRS-authorized CA
- **SDK:** A2A MeF Software Development Kit (SDK) Toolkit available for developers

**A2A Technical Details:**
- Uses SOAP envelopes with ZIP containers
- Designed for "registered transmitters who send large complex returns"
- WS-I (Web Services-Interoperability) security standards
- WSDL version R10.9 for tax year 2025/processing year 2026
- Strong authentication required since January 2009
- **Publication 4164:** The developer bible — "Modernized e-File Guide for Software Developers and Transmitters"

**Key difference from CRA:** The IRS publishes XML schemas publicly. CRA keeps T1 specs under NDA.

### IRS Direct File (Open Source!)

**The IRS released their tax filing software as open source:**
- **GitHub:** https://github.com/IRS-Public/direct-file
- **License:** Public domain (US government work)
- **Architecture:** Interview-based web app -> generates MeF XML -> transmits via MeF API
- **Also provides:** State API with MeF XML + enriched JSON format

**2026 Status:** IRS told 25 partner states that "Direct File will not be available in Filing Season 2026." No future launch date set. However, the open-source code remains available for states and developers.

**Derivative:** https://github.com/openfiletax/openfile — "A free tax tool based on IRS Direct File"

> **ATLAS NOTE:** The IRS Direct File codebase is a goldmine. It shows exactly how to generate MeF-compliant XML and transmit to the IRS. If CC ever needs to file US taxes (dual citizenship scenario), this is the playbook.

### IRS IRIS (Information Returns Intake System)

For 1099s and information returns:
- **URL:** https://www.irs.gov/filing/e-file-information-returns-with-iris
- **Free:** Yes, completely free
- **Two channels:** Taxpayer Portal (UI) and A2A (programmatic API)
- **API:** REST-based with access tokens
- **Supports:** All Forms 1099 for tax year 2022+
- **TCC:** Need a 5-digit Transmitter Control Code
- **Publication 5718:** Processing Year 2026 IRIS guide

### US vs Canada Comparison

| Feature | CRA (Canada) | IRS (US) |
|---------|-------------|----------|
| Public API | No (proprietary) | Yes (MeF A2A + IRIS) |
| XML schemas published | Info returns only | All forms |
| Open-source filing tool | CDS prototype (archived) | Direct File (full, public domain) |
| Programmatic filing | Via certified software only | Via A2A SOAP web service |
| Developer access | NDA + certification | Public schemas + SDK |
| Cost to develop | Certification process | Free (schemas public) |

**The US is years ahead of Canada for programmatic tax filing.**

## 7. Receipt Tracking & OCR Automation

### HubDoc (Intuit-Owned)

- **URL:** https://www.hubdoc.com/
- **Cost:** ~$12/month, unlimited documents
- **OCR:** Yes — extracts supplier, amount, invoice number, due date
- **Integrations:** QuickBooks Online, Xero (NOT QuickBooks Desktop)
- **API:** **NO API** — no programmatic access whatsoever
- **Automation:** Auto-fetch from banks/suppliers, email forwarding
- **Limitations:** Cannot integrate with QuickBooks Desktop, no developer access

### Dext (formerly Receipt Bank)

- **URL:** https://dext.com/
- **Cost:** From $37.50/month (250 documents)
- **OCR:** 99% accuracy, supports handwritten documents
- **Line item extraction:** Product names, quantities, prices, tax details
- **Integrations:** QuickBooks Online, Xero, Sage
- **API:** **YES — REST API available**
  - **Base URL:** `https://api.precision.dext.com/`
  - **Auth:** Bearer token (generate in Practice Settings > API Tokens)
  - **Rate limit:** 60 requests/minute
  - **Endpoints:**
    - `GET /clients` — list all clients
    - `GET /clients/{clientId}` — detailed client info
    - `GET /clients/{clientId}/activity-stats` — rolling annual stats
  - **Receipt submission:** Send images/PDFs, receive structured data back
  - **Full docs:** https://help.dext.com/en/articles/105818-precision-api
- **Best for ATLAS:** Dext has the most accessible API for receipt processing

### Shoeboxed

- **URL:** https://www.shoeboxed.com/
- **API:** **YES — HTTP web service (API v2)**
  - **GitHub:** https://github.com/Shoeboxed/api
  - **Webhooks:** Notifications for document status changes ("created", "processed")
  - **Auth:** HMAC SHA-256 signatures
  - **Legacy API:** Being deprecated, migrate to v2
  - **Contact:** api@team.shoeboxed.com
- **Integrations:** QuickBooks, Xero, Evernote, Bench
- **Cost:** Varies by plan

### Custom OCR Pipeline (DIY — Recommended for ATLAS)

The most powerful and cost-effective approach for ATLAS:

**Google Cloud Vision API:**
- OCR accuracy: Very high for printed receipts
- Cost: First 1,000 units/month free, then $1.50/1,000 units
- Can extract raw text from receipt images
- **URL:** https://cloud.google.com/use-cases/ocr

**OpenAI Vision API (GPT-4o with vision):**
- Can understand receipt layout, not just OCR text
- Extracts vendor, date, items, amounts, tax, total
- Can categorize expenses against chart of accounts
- Cost: ~$0.01-0.03 per receipt (depending on image size)

**Recommended Pipeline:**
```
Receipt Photo/PDF
    -> Google Cloud Vision API (OCR text extraction)
    -> OpenAI GPT-4o (structured data extraction + categorization)
    -> JSON output: {vendor, date, items[], subtotal, tax, total, category, t2125_line}
    -> QuickBooks API or local database
```

## 8. Atlas Assessment — Path to Automated AI Accountant

### The Reality Check

**Filing taxes programmatically in Canada is hard because:**
1. CRA does NOT publish T1/T2 XML schemas publicly (unlike IRS)
2. The `.tax` file format is under NDA for certified developers only
3. There is no public REST API for CRA tax filing
4. All filing must go through CRA-certified software
5. Annual recertification is required

**Filing taxes programmatically in the US is easier because:**
1. IRS publishes all XML schemas publicly
2. MeF A2A provides a real SOAP web service
3. IRS Direct File source code is open source and public domain
4. IRIS provides REST API for information returns

### Recommended Strategy for ATLAS

#### Phase 1: Immediate (2026 Tax Year)

**Goal:** Automate everything EXCEPT the final CRA submission

1. **Receipt OCR:** Build n8n workflow with Google Vision + Claude for receipt processing
2. **Bookkeeping:** Use QBO API for expense tracking and categorization
3. **Tax calculation:** ATLAS calculates T1, T2125, Schedule 3 amounts using docs/ knowledge base
4. **Data preparation:** Generate structured JSON/CSV with all return data
5. **Filing:** Manually enter into Wealthsimple Tax or StudioTax (free), NETFILE

**Cost:** ~$0 (free software + existing API credits)

#### Phase 2: Professional Setup (2027)

**Goal:** Batch filing capability for CC + potential clients

1. **Register OASIS as EFILE service provider** (free, ~30 days)
2. **Get TaxCycle license** ($2,440/year) for batch EFILE
3. **ATLAS prepares data** -> TaxCycle ingests -> batch EFILE to CRA
4. **Receipt pipeline:** n8n + Google Vision + Claude -> QBO
5. **Can file T1, T2, T3** returns through TaxCycle

**Cost:** ~$2,440/year for TaxCycle

#### Phase 3: Product Path (2028+)

**Goal:** ATLAS as sellable CFO-in-a-box with tax filing

**Option A — Wrapper approach (easier):**
- ATLAS as AI layer that prepares data for certified software
- Like TaxGPT — automates existing tax software via UI automation
- Does not need CRA certification
- Sell as "AI tax preparation assistant"

**Option B — CRA certification (harder, more valuable):**
- Apply for CRA software developer program
- Sign confidentiality agreement
- Build NETFILE/EFILE-compliant engine
- Certify annually
- Sell as "AI-powered tax filing software" — first of its kind
- **Moat:** CRA certification is a significant barrier to entry

**Option C — US-first via IRS MeF (fastest to market):**
- Use public IRS schemas + Direct File open source code
- Build MeF-compliant filing engine
- Register as e-file provider with IRS
- File US returns programmatically
- **Then** apply for CRA certification for Canada
- Bigger market (330M vs 40M population)
