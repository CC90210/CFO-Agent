# Atlas System & Multi-Agent Skill

## When to activate
Any question about: how Atlas works, system, architecture, Bravo, CEO agent, pulse, multi-agent, Telegram, bot, infrastructure, PM2, commands, tools, capabilities, file structure.

## Architecture Overview
```
Atlas (CFO) ←→ Bravo (CEO)
    ↓                ↓
cfo_pulse.json   ceo_pulse.json
```

- **Atlas** = CFO + Research Analyst (`CFO-Agent/`)
- **Bravo** = CEO + Revenue Ops (`Business-Empire-Agent/`)
- Cross-agent data flows ONLY through pulse files. Neither agent writes to the other's directory.

## Key Architecture Files
- `brain/AGENT_ORCHESTRATION.md` — the Atlas↔Bravo contract + multi-runtime rules
- `brain/CAPABILITIES.md` — auto-generated capability registry (regenerated by `scripts/build_capabilities.py`)
- `brain/BRAIN_LOOP.md` — session start/end protocol
- `brain/INTERACTION_PROTOCOL.md` — how Atlas talks to CC

## Pulse Protocol
- Atlas publishes: `data/pulse/cfo_pulse.json`
- Bravo publishes: `C:\Users\User\Business-Empire-Agent\data\pulse\ceo_pulse.json`
- Freshness rule: >7 days stale = flagged
