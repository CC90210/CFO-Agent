# Trade Protocol — 10-Step Decision Framework

## Overview
Every trade passes through this 10-step protocol. No shortcuts. Capital is at stake. This protocol exists in core/trade_protocol.py but is documented here as the canonical reference.

## When to Use
- Every trade signal generated by any strategy
- Manual trade ideas from CC
- Position management decisions (add, reduce, exit)

## The 10 Steps

### Step 1: REGIME CHECK
**Question:** What is the current market regime?

1. Query regime_detector.py for current classification
2. Check regime for the specific asset (not just BTC)
3. Match regime to strategy suitability:
   - BULL_TREND: trend-following strategies active, mean reversion reduced
   - BEAR_TREND: short-biased strategies, defensive sizing
   - CHOPPY: mean reversion strategies preferred, trend-following reduced
   - HIGH_VOL: reduce all position sizes by 50%, widen stops
4. If regime is hostile to the strategy type: REJECT signal

### Step 2: SIGNAL GENERATION
**Question:** What is the signal and how strong is it?

1. Strategy generates raw signal (long/short/neutral)
2. Calculate conviction score (-1.0 to 1.0)
3. Minimum threshold: |conviction| >= 0.3
4. Record: strategy name, timeframe, signal type, raw conviction

### Step 3: CONFLUENCE CHECK
**Question:** Do multiple signals agree?

1. Check other strategies on the same asset
2. Check multiple timeframes (15m, 1H, 4H, 1D)
3. Confluence scoring:
   - 1 strategy agrees: conviction × 1.0 (no boost)
   - 2 strategies agree: conviction × 1.2
   - 3+ strategies agree: conviction × 1.5
4. Contradicting signals: reduce conviction by 0.5

### Step 4: RISK ASSESSMENT
**Question:** Can we afford this trade?

1. Check portfolio heat (total open risk as % of equity)
2. Check daily loss limit (5% — any remaining budget?)
3. Check max drawdown (15% — how much room?)
4. Check correlation with existing positions
5. If ANY limit would be breached: REJECT

### Step 5: POSITION SIZING
**Question:** How much?

1. Run position-sizing skill (Phase 1-6)
2. Apply strategy-specific modifier
3. Apply conviction modifier (high/medium/low)
4. Apply correlation adjustment
5. Final size must pass all validation checks

### Step 6: TIMING
**Question:** When to enter?

1. Check market hours (for forex/metals — avoid low-liquidity periods)
2. Check for upcoming high-impact events (FOMC, CPI, NFP)
3. Check spread (if unusually wide, wait)
4. Limit order vs market order decision:
   - High conviction + momentum: market order
   - Medium conviction + no rush: limit order at better price
5. Order timeout: 30 seconds for market, 4 hours for limit

### Step 7: EXIT PLAN (SET BEFORE ENTRY)
**Question:** Where do we get out?

1. Stop-loss: based on strategy type and ATR
   - Trend following: 3x+ ATR (wide)
   - Mean reversion: 1.5-2x ATR (tight)
   - Breakout: 2x ATR (medium)
2. Take-profit: based on risk-reward ratio
   - Minimum 2:1 reward-to-risk
   - Trend following: trailing stop (no fixed target)
   - Mean reversion: fixed target at reversion level
3. Trailing stop method: per-strategy from trailing_stop.py
4. Time-based exit: if position stagnates for >X bars (strategy-specific)

### Step 8: EXECUTE
**Question:** Place the trade.

1. Submit order via OrderExecutor (core/order_executor.py)
2. Verify fill (check order status after timeout)
3. Set stop-loss order immediately after fill
4. Record trade in database
5. Alert CC via Telegram (if configured)

### Step 9: MONITOR
**Question:** Is the trade behaving as expected?

1. Trailing stop updates on every tick
2. Break-even promotion: move stop to entry after 1:1 reached
3. Profit-lock tiers: lock 50% profit after 2:1
4. Regime change check: if regime flips, tighten stop
5. Correlation check: if new correlated position opened, review

### Step 10: POST-MORTEM
**Question:** What did we learn?

1. Was the entry correct? (price action confirmed signal?)
2. Was the sizing appropriate? (too large = uncomfortable, too small = irrelevant)
3. Was the exit optimal? (left money on table? or got stopped perfectly?)
4. Would a different stop-loss method have worked better?
5. Record: outcome, lesson, confidence adjustment
